# 授权测试中危优先 Skill：快速可报告成果

本 Skill 用于明确授权的 Web/API、外部暴露面、后台系统和漏洞赏金测试中，优先寻找“中危可确认、低影响、能快速形成报告”的漏洞。目标不是冲高危，也不是完整覆盖式审计，而是在有限时间内稳定产出可复现、可评级、可修复的中危发现。

本 Skill 的评级口径必须服从 `agent/skills/漏洞评级.md`：快速出成果不等于放宽中危门槛。任何发现都要先证明对厂商核心业务、真实用户权益、资产安全、数据机密性/完整性/可用性存在实际影响；只能证明配置异常、公开暴露、理论可利用或前置条件的线索，不计入中危成果。

## 适用场景

- 用户要求“中危优先”“快速出成果”“先拿可报告漏洞”。
- 目标时间有限，需要从大量线索中压缩出 1-3 个最可能确认的中危方向。
- 已授权 Web/API 测试、SRC/赏金范围内验证、阶段性快速摸底。
- 不适用于未授权目标、破坏性测试、真实用户数据批量读取、高强度爆破或追求 shell 的专项测试。

## 必读依赖

- 先读 `agent/skills/security-testing.md`：授权边界、证据要求、低影响原则。
- 涉及评级时读 `agent/skills/漏洞评级.md`。
- Web/API 目标使用 `agent/skills/security-testing.md` 的证据驱动、动态信息收集、漏洞发现链和影响边界停止规则。
- 需要细分方法时读 `agent/skills/hack-skill.md`，只加载当前现象匹配的 HackSkills 文件。

## 成果定义

优先追求以下中危成果，但必须能闭合实际危害：

- 未授权访问后台、管理 API、用户资料、订单、附件、导出数据等非公开资源。
- 水平越权、垂直越权、IDOR/BOLA，少量已知样本即可证明。
- 任意文件读、下载、预览、导出路径控制，能读到配置、日志、非公开附件或跨用户文件。
- 存储型 XSS、可稳定触发的认证态 XSS、后台可见 XSS。
- 一般 SQL 注入，至少证明数据库名、用户名、表结构或稳定布尔/报错差异。
- 敏感配置泄露，如 DB 连接串、Secret、AccessKey、JWT 密钥、可用凭据、非公开源码包；source map 或前端源码只有在包含敏感凭据、内部非公开业务逻辑并能支撑实际影响时才作为中危候选。
- 找回密码、验证码、OAuth、JWT、Session 绑定缺陷导致账号风险但未达到高危。
- 后台弱口令、默认口令、弱认证入口；不包括普通前台弱口令。
- CORS、Host Header、Open Redirect 等只有在能组合敏感接口、凭据或认证流程时才作为中危候选。

以下发现默认不能作为中危成果，只能作为信息项、低危观察、加固建议或组合链材料，除非补充证据能闭合实际影响：

- 不涉及敏感信息的 CORS、危险 HTTP 方法声明、安全响应头缺失、TLS/Server 头和普通版本号暴露。
- 公开后台入口、公开 Swagger/OpenAPI、health/config 元数据、默认错误页、静态前端配置、source map 或前端源码泄露，但未证明敏感读取、写入、认证绕过或业务影响。
- 公开栏目、公开附件、公示文件、政策文件、统计表、办事指南等预期公开内容。
- 示例文件、无敏感信息的日志/内网 IP/内存信息泄露、地图 Key 或其它无资源限制证明的 Key。
- 虚假的前端绕过、上传 PDF 导致的 XSS、无敏感接口的未授权访问、无利益影响的点赞/收藏/转发并发问题。
- 仅有文字描述、扫描器命中、理论猜测或缺少完整复现细节的报告线索。

## 60 分钟路线

### 0-10 分钟：快速压缩攻击面

确认：

- 真实 DNS / Host / SNI / 入口 IP，避免 Fake-IP 误扫。
- 主站、API、admin、后台、对象存储、下载、上传、导出、Swagger/OpenAPI。
- 登录、注册、找回密码、验证码、OAuth、JWT、Cookie 行为。
- 前端 JS 中的接口前缀、对象 ID、用户/租户/机构/订单/附件字段。

优先保留：

- 大型 SPA、API 网关、后台入口、业务系统。
- 有对象 ID、附件、导入导出、文件预览、个人中心、订单、角色权限的系统。
- 明确版本产品、暴露调试接口、source map、swagger、配置文件线索。

快速降级：

- 纯静态站、营销页、安全头缺失、TLS 配置问题、无敏感接口的普通 CORS。
- 只有标题或 Server 头的版本猜测。
- 无认证、无数据、无业务对象的低价值路径。
- 公开栏目和公开附件；没有隐藏路径、对象枚举、未授权越权、批量导出或超出公开目的证据时不进入中危队列。
- 只证明存在 Swagger、source map、前端配置、默认错误页或健康检查，但没有敏感字段、凭据、非公开数据或可利用业务动作。

### 10-30 分钟：中危高命中验证

按优先级推进：

1. 越权：正常请求 -> 去 Cookie/Token -> 换对象 ID -> 换用户/租户/机构字段 -> 低权限访问管理接口。
2. 文件：download/export/preview/file/path/url/id 参数 -> 路径穿越 -> 跨用户附件 -> 配置/日志/source map。
3. 认证：登录/注册/找回密码/验证码 -> 账号枚举、验证码复用、token 泄露、登录态提前下发。
4. 信息泄露：`.map`、备份包、配置、swagger、调试接口、Git 泄露、对象存储公开读；必须继续证明敏感内容、非公开性、可用凭据、认证绕过或业务影响。
5. 注入：只做低影响报错、布尔、类型差异；SQLi 至少证明数据库名/用户名。
6. XSS：优先存储点、后台可见点、富文本/预览/文件名/导入导出链路。
7. 弱认证：后台默认口令或弱口令只做小样本、低频、授权范围内验证。

### 30-45 分钟：组合和确认

- 把信息泄露、认证缺陷、对象 ID、后台入口、文件下载点组合验证，不孤立看单点。
- 优先补齐一条最稳中危的对照证据，而不是扩散到大量未确认线索。
- 对 CORS、Open Redirect、Host Header、Swagger、source map、公开配置、版本信息等弱证据线索，先寻找敏感接口、凭据、认证绕过、非公开数据读取或可控业务动作；找不到闭环时降为低危/信息项。
- 如果需要第二账号、真实验证码、测试数据或低权限后台账号，停止扩大影响并记录人工配合事项。
- 如果某条路径已明显接近高危，切换到 `agent/skills/aggressive-high-impact.md` 或 `agent/skills/security-testing.md` 的影响边界停止规则。

### 45-60 分钟：成果固化

输出最强 1-3 条成果：

```text
结论：
等级：
已证明事实：
关键请求/响应：
影响范围：
为什么达到中危：
为什么未升级高危或降为低危：
为什么不是无实际危害/证据不足：
还差什么证据：
下一步需要的账号/样本/授权：
```

## 证据标准

每个中危候选必须记录：

- 目标、接口、认证状态、请求方法、路径、关键参数。
- 对照请求：正常、未认证、低权限、替换对象 ID 或替换角色后的差异。
- 响应状态码、关键字段、可证明影响的数据片段。
- 影响范围：单用户、跨用户、后台、租户、机构、全站。
- 是否需要账号、第二账号、验证码、测试数据或追加授权。
- 为什么达到中危，为什么没有升级高危或降为低危。
- 对照 `漏洞评级.md` 说明为什么不是预期公开内容、理论风险、弱证据风险、无敏感信息配置缺陷或只能作为组合链材料。

## 停止规则

- 已用最小样本证明中危后，停止扩大读取、枚举和写操作。
- 发现高危前置条件时，切换到 `security-testing.md` 或 `aggressive-high-impact.md` 的停止规则，不继续拿 shell、不写 webshell、不改配置。
- 写操作必须有明确授权、可回滚、测试数据；没有条件时只写验证计划。
- 不把公开栏目、公开附件、公示文件直接包装成漏洞，除非能证明未授权、隐藏路径、对象枚举或批量导出缺陷。
- 不为了快速出成果夸大扫描器结果；扫描命中必须补请求/响应、对照样本和业务影响。
- 不把无实际危害的现象计入中危数量；没有闭合实际影响时，明确写为“受限未验证 / 低危观察 / 信息项 / 加固建议”。

## 结束前自检

- 是否至少覆盖 API、认证、对象 ID、文件、后台入口、敏感配置六类高命中面？
- 是否把所有低危/信息泄露线索尝试组合到中危路径？
- 是否已按 `漏洞评级.md` 过滤无实际危害、弱证据、预期公开内容和不予奖励类型？
- 是否存在需要第二账号、验证码、测试数据或追加授权才能确认的中危候选？
- 是否已把中危及以上发现写入阶段性报告或正式报告索引？
