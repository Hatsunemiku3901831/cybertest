# 授权测试高危优先 Skill：成果导向攻击面压缩

本 Skill 用于明确授权的 Web/API、外部暴露面、内网服务和上线前安全测试中，需要快速产出高影响结果的场景。目标是最大化发现高危/严重风险的概率，而不是生成完整覆盖式审计报告。

它不适用于未授权目标。CTF/靶场夺旗优先用 `ctf-web-flag-rush.md`；常规稳健报告优先用 `security-testing.md` 和 `hack-skill.md`。

## 工作目标

- 在有限时间内优先寻找可形成高危结论的攻击链。
- 把“线索很多”压缩成“最可能出高危的 1-3 条路径”。
- 优先验证能产生业务影响、权限突破、数据访问、执行风险或信任边界破坏的点。
- 不把低危硬化问题包装成高危；没有高危时明确给出最接近高危的阻塞点和下一步授权需求。

## 成果定义

优先追求以下成果：

- 未授权或低权限读取敏感业务数据。
- 水平越权、垂直越权、IDOR/BOLA。
- 任意文件读取、服务端请求伪造到内网敏感服务。
- 文件上传进入可执行/可处理危险链路。
- 任意 Origin 凭证化 CORS 与敏感 API 组合。
- 可控参数进入后端高风险处理器，例如文档转换、任务执行、签章、导入导出、模板渲染。
- 网关错误配置导致内部服务路由、服务名、管理面或下游业务暴露。
- 源站/入口 IP 绕过预期 CDN/WAF，并可访问真实业务。
- 高危 CVE 的组件暴露加版本/配置/行为证据。
- 内网服务暴露导致匿名读写、管理面访问、未认证数据库或弱认证入口。

## 激进但不乱打

允许更主动：

- 更快地从信息收集切换到验证。
- 更早使用扫描器、爬虫、目录/参数 fuzz、公开 PoC 信息检索。
- 对高价值路径做小样本写入、上传、权限边界验证，但必须可回滚并在授权范围内。
- 对发现的业务对象 ID 做少量、已知样本的越权验证。
- 对上传、转换、导入、任务执行等危险链路做边界探测。

仍然禁止：

- 未授权目标测试。
- 在生产目标上执行会导致崩溃或拒绝服务的 PoC。
- 为了“出高危”夸大没有行为证据的线索。

## 90 分钟高危优先路线

### 0-10 分钟：攻击面压缩

快速确认：

- 主机、vhost、入口 IP、TLS 证书、Server、技术栈。
- 登录态、API 前缀、对象存储域名、上传/下载域名。
- 前端 JS 中的大 API surface、管理路由、业务对象名。
- 版本明显的 OA、网关、中间件、CMS、Spring、IIS/ASP.NET、nginx、Tomcat。

优先保留：

- 有登录、有上传、有导入导出、有文件预览、有异步任务、有文档转换的系统。
- 大型 SPA API，而不是纯静态站点。
- 网关/API host，而不是只有营销页的 host。
- 带版本号的 OA/后台/中间件。

### 10-25 分钟：高价值端点定位

从 HTML/JS、crawl、proxy 流量、swagger/openapi、报错中提取：

```text
upload, import, export, download, preview, convert, task, job, sign, template,
file, attachment, order, contract, user, admin, role, permission, tenant,
callback, webhook, fetch, url, redirect, render, pdf, excel
```

优先测试：

- 上传和公开访问链。
- 下载和任意文件路径参数。
- 导入/转换/预览/模板渲染。
- 业务对象读接口和列表接口。
- 管理接口只读访问。
- CORS 预检和 cookie-only 行为。
- 网关路由、服务名泄露、管理路径。

优先工具：

- `tool/arjun_json.py`：发现隐藏参数和危险参数名。
- `tool/kiterunner_json.py`：发现 API/网关隐藏路由；当前本机 `kr` 未安装，暂不优先调用。
- `tool/rustscan_nmap.py`：压缩端口面并确认服务指纹。
- `tool/gitleaks_json.py`：源码/备份/泄露仓库 Secret 扫描。
- `tool/dalfox_json.py`：XSS/admin bot/preview 链候选验证。

### 25-55 分钟：冲高危攻击链

按优先级推进：

1. **越权链**：登录态接口 -> 去 token -> 换对象 ID -> 换租户/用户字段 -> 低权限访问管理接口。
2. **上传链**：危险后缀/MIME -> 公开预览 -> inline/nosniff -> API 信任对象域名 -> 后端处理器是否消费该对象。
3. **CORS 链**：任意 Origin -> credentials -> cookie-only 敏感 GET -> 无害写接口 -> 浏览器 proof。
4. **文件链**：download/preview/export 参数 -> path traversal/LFI -> 读配置 -> 拿数据库/密钥 -> 读敏感数据。
5. **SSRF 链**：fetch/webhook/render/url 参数 -> 127.0.0.1/内网/metadata -> 管理面或云凭证线索。
6. **网关链**：Host/SNI/path 组合 -> Java JSON 错误 -> service-discovery 503 -> 内部服务名 -> 下游路径重放。
7. **CVE 链**：版本/组件暴露 -> 非破坏 PoC 或行为证据 -> 配置条件 -> 影响面判断。
8. **内网服务链**：匿名 FTP/NFS/数据库/打印/管理面 -> 读写边界 -> 敏感文件或配置暴露。

### 55-75 分钟：扩大半径

如果主路径未出高危：

- 切到 sibling host、API host、admin host、对象存储 host。
- 从 JS/API 名称反推未测业务模块。
- 用小字典跑敏感路径，不做无意义全量爆破。
- 对版本化产品查公开 advisory 和默认路径。
- 对已知开放端口按服务族跑 safe scripts 或只读探测。

### 75-90 分钟：成果固化

把最强 1-3 条链写成：

```text
结论：
已证明事实：
关键请求/响应：
影响：
为什么是高危或接近高危：
还差什么证据：
下一步需要的授权/账号/样本：
```

如果没有高危，不要硬报。输出“最接近高危的路径”和阻塞点，例如缺第二账号、缺 disposable 业务对象、缺基础设施侧版本/配置、缺处理器可达样本。

## 高危优先判断表

| 线索 | 立刻深挖条件 | 高危成立条件 |
|---|---|---|
| 上传可公开访问 | HTML/SVG/危险后缀、inline、API 信任对象域 | 可影响认证态用户、进入后端处理器、或形成存储型攻击链 |
| 任意 Origin CORS | `Access-Control-Allow-Credentials: true` | 浏览器可带 cookie 读敏感 API 或执行无害写 |
| Java Gateway 503 | 泄露 service name、route family 未认证可达 | 下游服务实例可达、管理接口可达、或鉴权绕过 |
| 源站 IP 暴露 | 固定 Host/SNI 可访问真实业务 | 绕过预期 CDN/WAF 或直连访问敏感 vhost |
| 版本化 OA/CMS | 明确版本、插件、默认路径 | 匹配高危 CVE 且有非破坏行为证据 |
| 下载/预览参数 | `file/path/url/id` 可控 | 读到配置、源码、敏感对象或跨用户文件 |
| 导入/转换/任务 | 参数绑定进入业务处理 | 可控文件/URL/模板进入高风险处理器 |
| 内网匿名服务 | FTP/NFS/DB/管理面开放 | 可匿名读敏感文件、写入可影响服务、或访问管理数据 |

## 复盘沉淀出的优先级

从当前复盘看，最容易产出高影响结果的顺序：

1. 上传 + 公开对象 + CORS/浏览器信任边界。
2. 大型 SPA 的 API 越权、对象 ID、租户/用户字段。
3. 网关路由、Java 错误、service-discovery 泄露和下游可达性。
4. 源站/入口 IP 暴露与固定 Host/SNI 业务访问。
5. 版本明确的 OA/中间件/CVE 暴露，但必须补行为或配置证据。
6. LAN 中匿名 FTP/NFS/打印/管理面和弱隔离。
7. 静态站点硬化问题通常低产，除非找到后台、下载、源码、配置或表单处理器。

## 常见浪费时间点

快速跳过或降优先级：

- SPA fallback `200 text/html`。
- 只有 `Server` 头或版本猜测。
- 没有敏感接口的普通 CORS。
- 只有安全头、TLS、mixed content。
- 组件路径、静态资源路径、前端路由误报。
- 全端口扫描拖很久但已有 top-port/risk-port 可用证据。
- 大量 404/405/timeout 没有响应差异。
