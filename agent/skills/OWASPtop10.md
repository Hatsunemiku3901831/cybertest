# OWASP Top 10 Web Security Testing Skill

本文件用于 Codex 在已授权 Web/API 安全测试中快速按 OWASP Top 10:2025 建模、验证和记录证据。它是项目内工作指引。

资料基线：

- OWASP Top Ten Web Application Security Risks 项目页说明当前发布版本为 OWASP Top Ten 2025。
- OWASP Top 10:2025 官方列表包含 A01 Broken Access Control 到 A10 Mishandling of Exceptional Conditions。
- 官方入口：https://owasp.org/www-project-top-ten/ 和 https://owasp.org/Top10/2025/


## 快速流程

1. 确认范围：域名/IP、路径、API、账号角色、允许的测试动作、禁止事项。
2. 建立资产地图：页面、接口、参数、对象 ID、认证态、角色、上传点、回调点、后台功能。
3. 建立基线请求：保存正常认证态请求、未登录请求、低权限请求和管理员请求的差异。
4. 按 A01-A10 逐项测试，先做高概率和高影响风险。
5. 每个疑点只用少量样本验证，确认后停止扩大影响。
6. 输出报告：复现步骤、证据、影响、风险等级、修复建议、验证状态。

## A01:2025 Broken Access Control

核心风险：用户能越过权限边界读取、修改或执行本不该访问的对象或功能。

常见场景：

- 未登录可访问需要登录的 API、文件、个人信息、后台页面。
- 水平越权：用户 A 访问或修改用户 B 的订单、成绩、资料、工单、附件。
- 垂直越权：普通用户访问管理接口、运营功能、审核功能、导出功能。
- IDOR：只替换 `id`、`userId`、`recordId`、`shopId`、`machineId` 即可访问对象。
- 只在前端隐藏菜单，后端没有校验角色。
- 强制浏览 `/admin`、`/api/admin/*`、导出接口、调试接口成功。

低影响测试方法：

- 去掉 Cookie、Authorization、CSRF token 后重放只读请求，期望 401/403 或登录跳转。
- 用低权限账号访问高权限只读接口，比较状态码和敏感字段。
- 用两个授权测试账号替换对象 ID，验证是否跨账号读取。
- 对写接口先检查 OPTIONS、路由、错误信息；需要写入时使用可回滚测试对象。

证据要点：

- 原始用户 A 请求和响应。
- 替换身份或对象后的请求和响应。
- 能证明越权的数据字段，如邮箱、手机号后四位、昵称、订单号、角色字段。
- 后端是否返回 200、业务成功码或敏感字段。

修复建议：

- 所有敏感对象由后端基于会话主体重新计算归属，不信任客户端传入的用户或角色。
- 在 controller/service/repository 层统一做对象级授权。
- 管理接口按角色和操作粒度校验。
- 默认拒绝，显式允许；权限失败返回一致的 401/403。

## A02:2025 Security Misconfiguration

核心风险：默认配置、调试配置、错误暴露或安全头缺失导致攻击面扩大。

常见场景：

- 调试页面、Swagger、GraphQL playground、Actuator、Prometheus、pprof、phpinfo 暴露。
- 目录索引、备份文件、`.git`、`.env`、日志、源码包可下载。
- CORS 过宽，如任意 Origin 携带凭据。
- 错误页泄露栈、SQL、内部路径、环境变量、版本号。
- 安全响应头缺失或配置弱：HSTS、CSP、X-Frame-Options、X-Content-Type-Options。
- 默认账号、默认密钥、示例配置未删除。

低影响测试方法：

- 枚举常见调试路径和静态敏感文件，只做 GET/HEAD。
- 检查响应头、错误页、重定向、跨域策略。
- 对不存在路径、错误参数触发普通错误，观察是否泄露内部信息。
- 检查 TLS、Cookie 属性和缓存策略。

修复建议：

- 生产环境关闭调试、目录索引和交互式控制台。
- 管理、监控、调试接口加网络隔离和强认证。
- 统一错误处理，外部只返回必要信息。
- 使用基线安全头，限制 CORS allowlist。
- 清理默认账号、示例密钥、备份文件和构建残留。

## A03:2025 Software Supply Chain Failures

核心风险：依赖、构建、镜像、插件、第三方脚本或更新链路被污染。

常见场景：

- 依赖存在已知 CVE，版本长期未更新。
- 使用来源不明的包、镜像、脚本、浏览器 CDN 资源。
- 构建流程未锁定版本，依赖投毒或混淆包名可进入生产。
- CI/CD token 权限过大，制品发布缺少签名和审批。
- 更新包、补丁、模型、规则文件缺少完整性校验。

低影响测试方法：

- 读取 lockfile、Dockerfile、CI 配置、镜像 tag、前端 CDN。
- 使用项目已有审计命令或离线清单比对 CVE。
- 检查是否 pin 版本、校验 hash、启用签名和 provenance。
- 检查构建日志和发布流程是否暴露 secret。

修复建议：

- 使用 lockfile、固定基础镜像 digest、限制第三方源。
- 对依赖、镜像、制品做 SBOM 和漏洞扫描。
- 对发布制品签名，更新包校验 hash/signature。
- 最小化 CI token 权限，隔离构建和部署权限。

## A04:2025 Cryptographic Failures

核心风险：敏感数据在传输、存储或处理时未被正确保护。

常见场景：

- HTTP 明文传输登录、token、个人信息、下载链接。
- 密码使用弱哈希或无 salt，如 MD5/SHA1/plaintext。
- token、验证码、重置链接熵不足或过期时间过长。
- Cookie 缺少 `Secure`、`HttpOnly`、`SameSite`。
- 机密信息写入日志、前端、localStorage、URL 参数。
- TLS 配置弱、证书错误、允许过时协议。

低影响测试方法：

- 检查登录、重置、支付、个人资料接口是否强制 HTTPS。
- 检查 Cookie、缓存头、敏感字段是否出现在 URL、日志或响应。
- 对 token 只做格式、长度、过期、绑定关系检查，不批量猜测。
- 如果有源码权限，检查密码哈希、密钥管理、随机数来源。

修复建议：

- 全站 HTTPS，启用 HSTS，禁止敏感操作降级。
- 密码使用 Argon2id、bcrypt 或 PBKDF2，并配置合理成本。
- token 高熵、短有效期、服务端可撤销，绑定关键上下文。
- 机密使用 KMS/secret manager，不写入源码、镜像和日志。

## A05:2025 Injection

核心风险：不可信输入被解释器当作命令、查询、模板或表达式执行。

常见场景：

- SQL/NoSQL/LDAP/XPath 注入。
- OS command injection。
- Server-side template injection。
- Header、日志、CSV、公式注入。
- GraphQL 查询复杂度滥用或字段级注入。
- 表达式语言、反序列化参数、搜索语法注入。

低影响测试方法：

- 优先使用无破坏的布尔、类型、语法错误和延时阈值极低的样本。
- 对搜索、登录、筛选、排序、导出、回调、文件名等输入点逐项检查。
- 比较正常输入、特殊字符、边界类型、数组/对象参数的响应差异。
- 禁止执行真实系统命令、删除、写文件、外带数据。

安全样本思路：

- SQL：单引号、括号、布尔条件、排序字段非法值，观察语法错误或逻辑绕过。
- Command：分隔符字符只用于确认是否被拒绝或转义，不执行实际命令。
- Template：普通表达式字符用于确认是否原样输出，不访问敏感对象。
- CSV：以 `=`, `+`, `-`, `@` 开头的字段检查导出时是否转义。

修复建议：

- SQL/NoSQL 使用参数化查询或安全 ORM，不拼接查询。
- 命令调用使用参数数组和 allowlist，避免 shell。
- 模板只渲染可信模板，用户输入作为数据转义。
- 对排序、字段名、操作符使用 allowlist。
- GraphQL 限制深度、复杂度、字段权限和 introspection 暴露。

## A06:2025 Insecure Design

核心风险：系统设计本身缺少安全控制，无法靠局部修补完全解决。

常见场景：

- 业务流程可绕过：未支付发货、未审核发布、重复领取奖励。
- 缺少服务端状态机，客户端可任意传入状态。
- 重要操作无二次确认、无风控、无速率限制。
- 邀请码、优惠券、二维码、ticket 可预测或可重放。
- 多租户隔离模型不清晰。
- 威胁模型没有覆盖内部人员、供应链、回调伪造。

低影响测试方法：

- 画业务状态机：创建、审核、支付、绑定、领取、删除、导出。
- 尝试跳步骤、重复提交、乱序提交、修改金额/状态/角色等测试对象字段。
- 使用少量测试数据验证是否由服务端重新计算关键结果。
- 检查幂等性、重放保护、一次性 token、过期和绑定。

修复建议：

- 建立威胁模型和滥用用例。
- 关键业务状态由服务端控制，不信任客户端状态。
- 重要操作加入幂等键、重放检测、审计、风控和审批。
- 多租户和对象归属作为基础域模型设计，而不是接口补丁。

## A07:2025 Authentication Failures

核心风险：认证、会话、密码恢复或身份绑定流程可被绕过或滥用。

常见场景：

- 登录接口缺少速率限制、锁定、风险检测。
- 默认密码、弱密码策略、可枚举用户名。
- Session fixation，登出后 token 仍有效。
- 重置密码 token 可预测、不过期、未绑定账号/设备。
- MFA 可绕过，备用码过弱或不失效。
- OAuth/OIDC state、redirect_uri、nonce 校验缺失。

低影响测试方法：

- 检查认证错误信息是否枚举账号。
- 用少量请求验证速率限制存在，不做爆破。
- 登录前后、登出后、改密后重放旧 token。
- 检查重置链接、验证码、OAuth state 是否一次性、短期、绑定上下文。
- 检查 Cookie 属性、session 轮换和并发会话策略。

修复建议：

- 统一认证错误信息，加入速率限制和风险检测。
- 登录成功、权限提升、改密后轮换 session。
- 重置 token 高熵、短期、一次性、绑定账号和用途。
- MFA 验证必须在服务端强制，敏感操作重新认证。
- 严格校验 OAuth/OIDC `state`、`nonce`、`redirect_uri`。

## A08:2025 Software or Data Integrity Failures

核心风险：软件更新、数据反序列化、CI/CD 或可信数据链路缺少完整性保护。

常见场景：

- 反序列化不可信对象导致代码执行或权限绕过。
- 自动更新包、插件、规则、模型未签名。
- Webhook、回调、消息队列事件未验签。
- 前端加载第三方脚本无 SRI 或 allowlist。
- CI/CD 流水线可被低权限提交触发敏感部署。

低影响测试方法：

- 检查上传、导入、消息、任务参数是否包含序列化对象。
- 检查 webhook 是否验证签名、时间戳、重放窗口。
- 检查更新包、插件、脚本是否有签名、hash、固定来源。
- 检查 CI/CD 权限边界和生产部署审批。

修复建议：

- 避免反序列化不可信对象；必要时使用安全格式和类型 allowlist。
- 所有更新包、插件、脚本、规则文件做签名和完整性校验。
- Webhook 使用 HMAC/签名、时间戳、nonce 和重放缓存。
- CI/CD 最小权限，受保护分支、审批、环境隔离。

## A09:2025 Security Logging and Alerting Failures

核心风险：攻击、越权、异常业务行为没有被记录、告警或响应。

常见场景：

- 登录失败、越权、管理操作、导出、改密、MFA 失败不记录。
- 日志缺少主体、对象、结果、来源 IP、request id。
- 日志记录敏感信息，如密码、token、验证码、完整身份证号。
- 安全告警无人处理，无法关联用户行为。
- 审计日志可被普通用户修改或删除。

低影响测试方法：

- 触发少量认证失败、权限失败、参数错误和后台操作，检查日志是否记录。
- 检查日志是否包含敏感字段。
- 检查 request id 是否贯穿前端、API、后台任务。
- 检查是否对高风险事件有告警规则和保留策略。

修复建议：

- 记录认证、授权、敏感数据访问、配置变更、导入导出和安全异常。
- 日志结构化，包含主体、动作、对象、结果、来源、关联 ID。
- 对 token、密码、验证码、密钥做脱敏或不记录。
- 安全日志独立存储、不可篡改、保留足够周期。

## A10:2025 Mishandling of Exceptional Conditions

核心风险：异常、失败、边界条件或降级路径处理不当，导致信息泄露、权限绕过或业务不一致。

常见场景：

- 服务异常时默认放行或返回过多调试信息。
- 支付、审核、绑定、发券等流程在超时/重试后状态不一致。
- 并发请求导致重复执行、重复领取、重复退款。
- 外部依赖失败时使用过期权限、旧配置或不安全默认值。
- 错误恢复任务重复消费消息或跳过校验。

低影响测试方法：

- 使用非法类型、缺失字段、超长字段、重复参数测试错误处理。
- 对测试对象做少量重复提交和并发提交，验证幂等和锁。
- 模拟外部依赖失败只能在授权测试环境做；生产环境只做代码和配置审计。
- 检查异常路径和正常路径是否执行同样的鉴权、校验和审计。

修复建议：

- 异常时 fail closed，禁止默认放行。
- 关键业务操作使用事务、幂等键、唯一约束和状态机。
- 错误响应统一，不泄露内部栈和配置。
- 重试、补偿、消息消费必须保持鉴权和完整性校验。

## 端到端检查清单

基础信息：

- 目标范围、账号角色、测试窗口、允许动作已确认。
- 页面、API、对象 ID、角色、上传点、导出点、回调点已枚举。
- 正常请求、未登录请求、低权限请求、管理员请求有样本。

认证和授权：

- 未登录访问敏感接口返回 401/403。
- 低权限访问管理接口返回 403。
- 跨账号对象访问失败。
- 登出、改密、权限变更后旧 token 失效或降权。

输入和数据：

- 输入点对特殊字符、类型错配、超长值有一致处理。
- 查询、命令、模板、导出、日志没有解释执行用户输入。
- 敏感数据不在 URL、日志、前端源码、缓存中泄露。

配置和供应链：

- 调试接口、默认文件、备份文件、目录索引不可访问。
- CORS、Cookie、安全头、TLS 配置符合生产要求。
- 依赖、镜像、脚本、插件、更新包来源可追溯并有完整性保护。

业务和异常：

- 关键状态不能由客户端任意指定。
- 重复提交、乱序提交、并发提交不会造成重复权益或状态错乱。
- 异常路径不绕过鉴权、校验、审计。
- 高风险事件有日志、告警和保留策略。



## 常用安全命令模式

这些命令只作为低影响验证思路，执行前必须替换为授权目标。

```bash
# 查看响应头和状态码
curl -k -i https://example.test/path

# 未登录重放只读接口
curl -k -i https://example.test/api/resource/123

# 携带测试账号 cookie 的只读请求
curl -k -i https://example.test/api/resource/123 -H 'Cookie: session=REDACTED'

# 检查 CORS 反射
curl -k -i https://example.test/api/me -H 'Origin: https://attacker.invalid'

# 检查常见调试路径，只做 HEAD/GET
curl -k -I https://example.test/.env
curl -k -I https://example.test/swagger-ui/
curl -k -I https://example.test/actuator/
```
